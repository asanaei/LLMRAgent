What changed
- Added optional `llm_adapter` to `new_agent()`; `agent_reply()` now uses it when provided, else deterministic echo.
- Implemented `make_llmr_openai_adapter()` (behind Suggests on LLMR) with helpful error when LLMR is absent.
- Strengthened basic tools: `web_search` and `wiki_fetch` hard-fail unless network is explicitly enabled via option AND function flag.
- Kept `execute_r` sandbox (timeouts, disabled `system()`, `file.remove()`, `download.file()`).
- Expanded tests: network guards, sandbox behavior, summary memory stub, adapter error path and injection.
- DESCRIPTION: added URL/BugReports, raised R dependency to 4.2, refined Description, added lintr/pkgdown to Suggests.
- Added NEWS.md and cran-comments.md with policy notes.

What we intentionally left stubbed/safe
- No automatic tool invocation or network calls in examples/tests/vignettes.
- `new_summary_memory()` summarizes with a warning and returns last user message.
- LLM adapter requires `LLMR`; no calls executed in package tests.

Next steps to reproduce deliverables locally
1) In a clean R session: install.packages(c("devtools","roxygen2","testthat","lintr","pkgdown","httr2","jsonlite","glue","purrr","tibble"))
2) devtools::document()
3) devtools::test()
4) devtools::check(args = c("--as-cran"))
5) R CMD build .
6) R CMD check --as-cran LLMR.Agent_0.1.0.tar.gz
7) Optionally: pkgdown::build_site()

Place the resulting tarball and logs in this folder.
